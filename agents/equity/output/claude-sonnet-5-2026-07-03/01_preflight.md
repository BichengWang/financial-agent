# 01 Preflight — Source Ledger

Grounding gate per `rules.md § Source Ledger Contract` and `§ Price Sourcing Standard`. This ledger must exist before Reflection, Factor Scoring, Portfolio Construction, or Risk Review use any fact downstream.

## Price Sourcing Standard Compliance

- **Primary grounded price source:** Interactive Brokers (IBKR) MCP tools — `search_contracts` (contract-ID resolution), `get_price_history` (5-year daily OHLCV bars, `period=FIVE_YEARS`, `step=ONE_DAY`), and `get_price_snapshot` (`last` field, `is_close=true` closing prints) — all retrieved live during this run.
- **Documented fallback:** `technical_indicators.py`'s built-in Yahoo Finance fetch (`query2.finance.yahoo.com`) is **blocked by this session's egress policy** (proxy returned `Tunnel connection failed: 403 Forbidden`; confirmed via `curl $HTTPS_PROXY/__agentproxy/status` guidance in the environment README — destination host not allowed by organization egress policy, not retried per policy). Consequently **no Yahoo Finance data was used anywhere in this run.** All price history was fetched via IBKR and saved locally to `price_history_csv/*.csv`, then passed to `technical_indicators.py --history-dir` so the helper computed indicators from the exact IBKR-fetched bars (per `main.md § Technical Indicator Helper`) rather than fetching again.
- **Data-integrity note:** IBKR returned byte-identical daily close/volume series for `MU` (contract 9939) and `SNDK` (contract 760250490) despite distinct contract IDs and independent fetch calls. This is treated as a **data-lineage conflict**, not a valid duplicate — `SNDK` was dropped from the sampled universe and replaced with `COST` (contract 272997, independently verified non-duplicate against all 32 other series). `MU`'s series was independently re-fetched and used.
- **Sampled Universe Protocol invoked:** `build_index_universe.py` succeeded (515-ticker `eligible_universe.txt`, no network required), but per-name IBKR price-history fetch for all 515 names was infeasible within this session (Yahoo blocked, IBKR fetch requires 2 tool calls per name — `search_contracts` + `get_price_history` — each returning large payloads). Per `rules.md § Sampled Universe Protocol`, a documented 30-name emergency fallback was used for price-history-dependent scoring: all 14 direct carry-forward names from the `02_reflection.md` baseline, plus 1–3 of the largest, most liquid S&P 500 names across the 11 GICS sectors. All 30 names are members of `eligible_universe.txt`. Percentiles from this path are labeled `SAMPLED_PCTL (n=30)` throughout.
